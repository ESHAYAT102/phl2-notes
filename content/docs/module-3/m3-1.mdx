# Why go for stateful approach

We must go for stateful approach when we need to accumulate the output. Meaning, when we want to store the previous data.
Example:

```js tab="code"
// Stateless

const counter = (amount) => {
  let count = 0;

  count = count + amount;

  return count;
};

console.log(counter(3));
console.log(counter(2));
```

```js tab="code"
// Stateful

const counter = {
  count: 0,

  add(amount) {
    this.count = this.count + amount;
  },

  print() {
    console.log(this.count);
  },
};

counter.add(2);
counter.add(3);

counter.print();
```

```js tab="result"
2; // Stateful
```

```js tab="result"
5; // Stateless
```

For stateless, the output will not be 5. But for stateful, the output will be 5, because it stores the previous data/output.
