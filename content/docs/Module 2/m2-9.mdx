# Denormalizing Data

In this module we need to denormalize a data. The normalized code:

```js
const users = [
  { id: 101, name: "Alice" },
  { id: 102, name: "Bob" },
  { id: 103, name: "Charlie" },
];

const posts = [
  { id: 1, userId: 102, title: "My first post" },
  { id: 2, userId: 101, title: "React Hooks" },
  { id: 3, userId: 101, title: "Data Structures" },
  { id: 4, userId: 103, title: "CSS is fun" },
  { id: 5, userId: 102, title: "Node.js streams" },
];
```

**Scenario**: You have an array of users and a separate array of posts. You have to create a new array of users where each user object contains a posts array of their own posts. Expected output should look like this:

```js
[
  { id: 101, name: 'Alice', posts: [ { id: 2, ... }, { id: 3, ... } ] },
  { id: 102, name: 'Bob', posts: [ { id: 1, ... }, { id: 5, ... } ] },
  { id: 103, name: 'Charlie', posts: [ { id: 4, ... } ] }
]
```

The code solution would be this:

```js
const postByUserId = posts.reduce((table, post) => {
  const { userId } = post;

  if (!table[userId]) {
    table[userId] = [];
  }

  table[userId].push(post);

  return table;
}, {});

const userWithPosts = users.map((user) => {
  return {
    ...user,

    posts: postByUserId[user.id] || [],
  };
});
```

As you can see, this code is prety similar to the previous one except in this one the lookup table sorts out the `posts` array of object. And then then `userWithPosts` pushes the `posts` array of object into the `users` array (technically).
The `userWithPosts` is actually not pushing the `posts` array but instead it's just copying it into another array, which is stored in itself.
