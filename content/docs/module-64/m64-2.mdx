---
title: Module 64.2
description: Variable Scope
---

A variable can't be accessed outside the function it's declared in. They're available to child functions.

We can think of scopes as layers, each nested function/process is a new layer on top of the previous one.
Each new layers (scopes) are smaller than the previous payer, so the previous layer kan wrap around it.

Example:

```go
package main

import "fmt"

func main() {
	hello := "hello world"
	fmt.Print(hello)
}
```

The `hello` variable is the first layer here
(though we can declare an upper layer variable outside the main function. But we're not gonna do that for now)

---

```go
package main

import "fmt"

func main() {
	hello := "hello world"
	fmt.Print(hello)

	func() { // [!code highlight]
		test := "testing variable" // [!code highlight]
		fmt.Print(test) // [!code highlight]
	}() // [!code highlight]
}
```

The nested function (`test`) is the smaller layer on top of the previous one.

---

So we can use the `hello` variable in the nested function (smaller layer) because it's in the larger layer,
which wraps around the smaller layer. But we can't access the `test` variable from the `main` function,
because it's in the smaller layer, which can't wrap around the other layer.
